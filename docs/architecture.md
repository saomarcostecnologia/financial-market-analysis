# Arquitetura

## Visão Geral

Este projeto utiliza uma arquitetura em camadas seguindo os princípios do Domain-Driven Design (DDD) e Clean Architecture.

## Camadas

1. **Domain Layer**: Contém as entidades de negócio e regras de domínio
2. **Application Layer**: Contém os casos de uso da aplicação
3. **Infrastructure Layer**: Implementações concretas de repositórios e serviços
4. **Interface Layer**: APIs, jobs e outros pontos de entrada
